展示 @chenglou/pretext 零 DOM 重排布局引擎在 Markdown 编辑器中的核心能力： 文本测量、光标定位、选区高亮、自动换行行号、虚拟化滚动。

1. 零 DOM 重排文本测量 核心特性

传统方案需要将文本插入 DOM 并触发浏览器 reflow 来获取宽高。 Pretext 通过 Canvas API 进行纯算术测量，完全零 DOM 操作。

2. Pretext 驱动的光标定位 CursorEngine

点击下方文本区域，光标位置完全通过 pretext 计算 —— xyToOffset(x, y) 将点击坐标转换为文本偏移量， offsetToPosition(offset) 将偏移量转换为像素坐标。 零 DOM 查询，零 getBoundingClientRect。

PreMarkdown 是一个高性能 Markdown 引擎。 它使用 @chenglou/pretext 进行零 DOM 重排的文本布局。 所有光标位置和选区矩形都通过纯算术计算得出。 试试点击这段文字的不同位置，拖拽选择文字！ 中文、English、日本語の混排テキストも正確に処理できます。

💡 Pretext 的 layout() 在 prepare() 完成后是 O(1) 纯算术，耗时通常 < 0.001ms。 DOM 方案每次都触发浏览器 reflow，耗时随文本量线性增长。

6. 增量文档布局 updateDocumentLayout

编辑文档时，updateDocumentLayout() 只重新计算变更的段落高度，复用未变更段落。 适用于实时编辑场景，性能接近 O(1)。
